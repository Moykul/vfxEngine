## Runtime Data Flow Summary

1. `VfxLevaControls` creates `levaConfig` with default values from `getVfxValues()`
2. `useControls(() => levaConfig)` creates `allVfxControls` (live Leva state)
3. `useEffect` syncs `allVfxControls` to context via `updateVfxSettings`
4. `finalVfxValues` combines transforms with `allVfxControls`
5. `VfxEngine` receives `finalVfxValues` and renders the particles

## File I/O Flow Summary

1. **Save**:
   - User clicks "Save Settings" button
   - `fileManager.saveJSON({ vfxSettings: allVfxControls })`
   - File downloads with current Leva control values

2. **Load**:
   - User clicks "Load Settings" button вҶ’ file input вҶ’ select file
   - `fileManager.loadJSON(file, (data) => { ... })`
   - If structured format (`data.vfxSettings`):
     - Filter out transforms вҶ’ `setAllVfxControls(vfxOnlySettings)`
   - If legacy format (root keys like `pCount`):
     - Filter out transforms вҶ’ `setAllVfxControls(legacyVfxSettings)`

## Note on Data Filtering

Transform properties like `positionX`, `positionY`, etc. are managed separately from the Leva controls. When loading files, these transform properties are filtered out before being passed to `setAllVfxControls` to avoid errors in Leva since these properties are not defined in the control schema.
